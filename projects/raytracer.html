A custom Bounding Volume Hierarchy takes renders that would run for hours down to under a minute.

Every pixel you see is an independently traced ray: primary hit → shadow ray → up to two reflection bounces, gamma-corrected. No rasterization, no triangles — pure ray-scene intersection.

/02 Why build a ray tracer by hand?

Ray tracing is the clearest way to prove you understand rendering at first principles: camera models, intersection geometry, shading math, texture sampling, and — most importantly — the data structures that make it all tractable. Everything here is implemented directly: the vector math, the file parsers, the acceleration structure, the thread scheduler.

The engine ingests any standard Wavefront OBJ scene with its MTL material library, so it renders real-world assets — not toy hardcoded scenes.

/03 Architecture

parser/

Full OBJ/MTL ingestion

Parses vertices, texture coordinates, per-vertex normals, face groups, and material definitions — including diffuse/specular maps — into an indexed triangle soup ready for tracing.

accel/

Bounding Volume Hierarchy

Median-split BVH over axis-aligned bounding boxes. Ray traversal skips entire subtrees on a missed box test, turning O(n) scene intersection into roughly O(log n).

shading/

Phong + textures

Ambient, diffuse, and specular terms with material colors sampled from texture maps via bilinear filtering. Barycentric interpolation of vertex normals gives smooth shading across coarse meshes.

render/

Multithreaded tiling

The image is divided into tiles dispatched across all available cores. Rendering is embarrassingly parallel — throughput scales nearly linearly with thread count.

/04 The BVH is the whole ballgame

A naive tracer tests every ray against every triangle. At 2048×2048 — over four million primary rays — against 330,000 triangles, that is more than a trillion intersection tests before shadows or reflections. The BVH collapses this by wrapping the scene in a binary tree of bounding boxes:

private Hit? Traverse(BvhNode node, Ray ray, float tMax) { if (!node.Bounds.Intersects(ray, tMax)) return null; // prune the whole subtree if (node.IsLeaf) return ClosestTriangleHit(node.Triangles, ray, tMax); Hit? left = Traverse(node.Left, ray, tMax); Hit? right = Traverse(node.Right, ray, left?.T ?? tMax); return right ?? left; }

Combined with tile-based multithreading, the result: scenes that ran for hours render in under 60 seconds.

/05 What it demonstrates

Rendering fundamentals

Camera ray generation, intersection geometry, illumination models, and texture sampling implemented from the math up — the same foundations behind production renderers.

Performance engineering

Spatial data structures, cache-conscious traversal, and parallel work scheduling — the difference between a correct renderer and a usable one.

Robust file ingestion

Handling real-world OBJ/MTL assets means tolerating the format's quirks: negative indices, mixed face arities, missing normals, and multi-material groups.
